# SUMO Traffic OS (Intelligent Traffic Light Control System)

An advanced urban traffic simulation and control platform built with Python, Eclipse SUMO (Simulation of Urban MObility), and TraCI. This project models an intersection control system that treats individual traffic light installations as independent concurrent threads, managing shared simulation states using Operating System synchronization primitives like Semaphores, Mutex Locks, and Conditional Events.

The system features an automated network assembly pipeline, quick-launch configurations for varying traffic densities, and a real-time data logging architecture backed by both a static plotting utility and an interactive web-based Streamlit dashboard.

---

## 🚀 Key Features

* **Concurrent Multi-Agent Architecture:** Individual traffic lights operate as decoupled background worker threads running concurrent execution loops.
* **OS-Level Synchronization:** Utilizes `threading.Semaphore` to regulate traffic phase switches safely, and `threading.Lock` to prevent critical section race conditions across multi-threaded TraCI calls.
* **Dynamic Adaptive Timing:** Implements a localized algorithm calculating real-time queue length differentials ($qA$ vs $qB$) to adaptively adjust signal phase allocations and step durations.
* **Automated Network Pipeline:** Uses `netconvert` to compile XML network geometries and route flows into unified binary simulation files.
* **Dual-Reporting Analytics Suite:** Offers static analytical plots via `matplotlib` and a live interactive data dashboard using Streamlit.

🛠️ Installation & Environment Setup
1. Prerequisites
Ensure you have Python 3.10+ and Eclipse SUMO installed on your local host environment.

2. Set System Environment Variables
The execution engine requires direct access to your local SUMO installation bin pathway. Set your SUMO_HOME environment variable:

Windows PowerShell: [System.Environment]::SetEnvironmentVariable('SUMO_HOME', 'C:\Program Files (x86)\Eclipse\Sumo', 'User')

Linux / macOS Bash: export SUMO_HOME="/usr/share/sumo"

(Note: Restart your active terminal environment after updating configuration lines to load system variables).

3. Install Dependencies
Instantiate your project dependencies via pip: pip install -r requirements.txt

🚦 How to Run the Pipeline
Step 1: Compile the Network Mesh
Before running the engine, synthesize your raw node, type, and edge files into a single network configuration: python build_network.py

Step 2: Run the Simulation Scenarios
You can run the main simulation file directly, or utilize the automated batch configurations to launch specific traffic configurations:

Run via Automated Batch Script (Recommended for Windows): 
.\run_final_light.bat   # For light traffic simulation
.\run_final_heavy.bat   # For heavy traffic congestion testing

Run Manually via Python Engine: python main.py

Step 3: Analyze Simulation Outputs
After the simulation successfully exports data into the output/ folder, you have two diagnostic options:

Generate Static PNG Reports: python plot_results.py
Launch the Interactive Cloud/Local Dashboard: streamlit run streamlit_app.py

🧠 Technical Architecture & Core Logic

Synchronization Primitives
Because SUMO's traci client instance operates fundamentally over a single-threaded TCP socket loop, multi-threaded worker configurations risk mutating states out-of-order. This architecture employs rigid synchronization gates:

traci_lock = threading.Lock(): Serves as a Mutual Exclusion (Mutex) primitive. Wraps all calls reading or writing simulation states (such as vehicle velocity, phase indices, and incoming queue values) to isolate critical sections.

tls_update_sem = threading.Semaphore(1): Acts as a binary counting gateway limiting concurrent thread updates over global phase arrays, preventing overlapping modifications to identical junctions.

Adaptive Optimization FormulaEvery thread tracking an intersection calculates queue volume across distinct traffic flows ($gA, gB$) by reading immediate lane coordinates where speeds drop below $0.1 \text{ m/s}$:

$$desired = \begin{cases} Phase\_A & \text{if } qA \geq qB \\ Phase\_B & \text{if } qA < qB \end{cases}$$

The phase duration resets dynamically depending on absolute congestion values:
$$Duration = \max(8 + \max(qA, qB), 20)$$
